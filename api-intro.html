什么是 API

API 是一种约定 — 它告诉一个程序：你可以向我请求什么、用什么格式、我会返回什么。

像餐厅的菜单

你（客户端）看菜单点菜，厨房（服务器）做菜给你。你不需要知道厨房怎么做菜，只需要知道能点什么。

像万能插座

不管你的电器是什么品牌，只要有标准插头，就能接入电网。API 就是软件世界的「标准插座」。

一句话总结： API 是一个「接口」，让不同的软件系统能够互相通信。当你用手机查看天气时，天气 App 就是通过 API 从气象服务器获取数据的。

SECTION 03

常见 类型

不同的场景使用不同类型的 API，以下是开发者最常接触的几种。

REST API

最流行的 Web API 风格。用 HTTP 方法（GET/POST/PUT/DELETE）操作资源，返回 JSON 数据。简单、直观、无状态。

GraphQL

一种查询语言。客户端精确指定需要哪些数据，服务器只返回这些。解决了 REST 的「过度获取」或「不足获取」问题。

WebSocket

建立持久连接，支持实时双向通信。适合聊天、在线游戏、股票行情等需要实时推送的场景。

gRPC

Google 出品的高性能 RPC 框架。使用 Protocol Buffers 序列化，比 REST 更快，适合微服务间通信。

! 初学者只需要先掌握 REST API，它是目前使用最广泛的 API 风格，90% 的场景都能胜任。

SECTION 04

实际 例子

看看 API 在真实世界中是怎么被调用的。

获取天气信息

用一个简单的 HTTP 请求就能获取北京的天气：

GET 请求 
// GET /api/weather?city=beijing HTTP/1.1
// Host: api.example.com
// Authorization: Bearer YOUR_API_KEY
200 OK 响应 
{
  "city": "北京",
  "temp": 28,
  "weather": "晴",
  "humidity": 45
}

调用 AI API

调用 Claude API 发送一条消息（Python 示例）：

Python 
import anthropic

client = anthropic.Anthropic(api_key="your-api-key")
message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "你好！"}]
)
print(message.content[0].text)

REST API 详解

REST 是目前最流行的 API 设计风格，理解它就理解了 API 的 80%。

四种基本操作

GET 获取数据

从服务器读取信息，不修改任何数据。相当于「查」。

GET /api/users/123

POST 创建数据

向服务器提交新数据。相当于「增」。

POST /api/users
{"name": "小明", "email": "..."}

PUT 更新数据

替换服务器上的已有数据。相当于「改」。

PUT /api/users/123
{"name": "小红"}

DELETE 删除数据

从服务器移除数据。相当于「删」。

DELETE /api/users/123

关键概念

端点（Endpoint）
服务器上的一个 URL 地址，比如 https://api.example.com/users。每个端点提供特定的功能。
认证（Authentication）
证明「你是谁」。通常通过 API Key 或 Token 放在请求头中：Authorization: Bearer xxx
状态码（Status Code）
服务器告诉你的结果：200 成功、400 请求错误、401 未授权、404 未找到、500 服务器故障。
速率限制（Rate Limit）
服务器限制你每分钟/每天能调用多少次。防止滥用，也保护服务器稳定。

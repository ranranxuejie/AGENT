SECTION 02

CLI 终端命令

在终端中启动和管理 Claude Code 的所有命令行选项。

基础启动

claude启动交互式会话
claude "提示"带初始提示启动
claude -p "提示"非交互模式（输出后退出）
claude -c继续最近一次对话
claude -r选择并恢复历史会话
claude -r <id>恢复指定会话
claude -v查看版本号
claude -h查看帮助信息

模型与努力级别

--model sonnet指定模型（sonnet / opus / haiku）
--effort high努力级别：low / medium / high / max
--fallback-model模型过载时的备选模型
--bare极简模式：跳过 hooks、插件、CLAUDE.md

会话管理

-n "名称"为会话命名，方便 /resume 识别
--session-id <uuid>使用指定会话 ID
--fork-session恢复时创建新会话分支
--from-pr <id>恢复关联 PR 的会话

权限与安全

--permission-mode plan先规划再执行
--allowedTools "Bash(git *)"预批准特定工具
--disallowedTools禁止特定工具
--max-budget-usd 5设置 API 花费上限（美元）

输出与集成

--output-format json输出格式：text / json / stream-json
--json-schema <schema>结构化输出验证
--ide自动连接 IDE
--add-dir ../lib添加额外目录访问权限

斜杠 / 命令

在 Claude Code 会话中输入 / 开头的命令，快速执行各种操作。已安装的 Skills 也会注册为斜杠命令。

内置命令

⚡ 核心命令
命令说明
/help查看所有可用命令和使用提示
/clear清空当前对话上下文，重新开始
/compact压缩对话以节省 token，保留关键信息
/config打开或修改 Claude Code 设置
/cost查看当前会话的 token 用量和费用
/doctor检查 Claude Code 安装健康状态
/init为当前项目初始化 CLAUDE.md 文件
/memory编辑 CLAUDE.md 记忆文件
/model切换当前会话的模型
/permissions查看和修改工具权限
/resume恢复之前的对话
/review审查 Pull Request
/status显示认证状态和会话信息
/vim切换 vim 编辑模式
/terminal-setup设置终端集成（启用 Shift+Enter 换行）
/bug向 Anthropic 报告 bug
/login登录 Anthropic 账号
/logout退出登录

Skill 注册的命令

每个已安装的 Skill 都会自动注册为斜杠命令。输入 / 即可查看完整列表。

/tavily-searchAI 优化的网页搜索
/tavily-extract提取网页内容为 Markdown
/tavily-crawl爬取网站内容
/openalex-paper-search学术论文检索
/semantic-scholarSemantic Scholar 论文搜索
/pdfPDF 文件读取与生成

/grill-me设计评审访谈
/find-skills发现并安装新 Skills
/self-improvement记录学习与错误

Skill 使用方式

/ 显式调用

用 / 前缀直接触发特定 Skill

/tavily-search 人工智能最新进展

SECTION 04

@ 引用与特殊语法

用 @ 引用文件和文件夹，让 Claude 精确理解你要操作的上下文。

@ 文件引用

语法说明
@filename.txt引用指定文件（触发自动补全）
@path/to/folder/引用整个目录
@src/file.py:10-20引用文件的指定行范围
@ + Tab浏览并交互式选择文件
✓ 最佳实践：在提示中用 @ 指向相关文件，如 "重构 @src/utils.ts 使用 async/await"，比笼统描述高效得多。

其他特殊语法

!command直接执行 shell 命令（前缀 !）
Ctrl+V粘贴剪贴板图片到对话中
拖拽文件将文件拖入终端窗口自动添加路径
Shift+Enter多行输入（需先运行 /terminal-setup）

上下文感知能力

自然语言交互

在终端或 VS Code 中直接用自然语言描述需求，Claude 自动理解上下文

代码上下文感知

Claude Code 自动读取项目结构和代码，无需手动解释项目背景

SECTION 06

CLAUDE.md 项目记忆

CLAUDE.md 是 Claude Code 的"项目大脑"——每次会话开始时自动读取，让 Claude 记住你的偏好和项目规范。

文件位置（按加载顺序）

范围路径
全局 所有项目~/.claude/CLAUDE.md
项目根目录<project>/CLAUDE.md
父目录../CLAUDE.md, ../../CLAUDE.md
子目录src/CLAUDE.md, tests/CLAUDE.md

最佳实践

保持简洁

每一行都会在每次对话中消耗 token，只写真正需要的内容

行为规则

写"始终做 X"和"绝不做 Y"类型的指令，如编码规范、测试命令

项目信息

包含构建命令、部署步骤、架构决策等关键上下文

用 /init 自动生成

运行 /init 让 Claude 从代码库自动生成初始 CLAUDE.md

记忆系统

🧠 Memory 文件
路径说明
~/.claude/memory/MEMORY.md全局记忆（跨项目持久化）
<project>/.claude/memory/项目级记忆目录
> 你：以后帮我写代码时不要加注释
~ Claude：已记住，后续代码生成将不再添加注释。
i 直接用自然语言告诉 Claude 你的偏好，它会自动保存到记忆文件中，后续对话自动生效。

SECTION 07

MCP 模型上下文协议

MCP 服务器为 Claude Code 扩展外部工具和数据源，连接数据库、API、文件系统等。

配置位置

~/.claude/settings.json全局配置
.mcp.json项目根目录（项目级）
--mcp-config启动参数（会话级）
--strict-mcp-config仅使用指定的 MCP 服务器

CLI 管理命令

命令说明
claude mcp add <name> <cmd>添加 MCP 服务器
claude mcp add --transport http ...添加 HTTP 类型的 MCP 服务器
claude mcp add-json <name> <json>从 JSON 添加 MCP 服务器
claude mcp add-from-claude-desktop从 Claude Desktop 导入配置
claude mcp list列出已配置的服务器
claude mcp get <name>查看服务器详情
claude mcp remove <name>移除服务器
claude mcp serve将 Claude Code 作为 MCP 服务器启动

配置示例

.mcp.json 
{
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["my-mcp-server"],
      "env": { "API_KEY": "xxx" }
    },
    "http-server": {
      "type": "http",
      "url": "https://mcp.example.com/mcp",
      "headers": { "Authorization": "Bearer xxx" }
    }
  }
}

SECTION 08

进阶 高级技巧

从后台 Agent 到 Git Worktree，解锁 Claude Code 的全部潜力。

后台 Agent

claude agents打开交互式 Agent 管理界面
claude agents --json以 JSON 列出活跃会话
--agent <name>使用指定 Agent 启动会话
--brief启用 Agent 与用户通信
i 后台 Agent 可以在你继续工作的同时执行长任务，每个 Agent 拥有独立的会话、模型和权限配置。

Git Worktree 隔离工作区

claude -w my-feature创建新的 git worktree 工作区
claude -w --tmuxWorktree + tmux 会话
--from-pr <id>恢复关联 PR 的工作
--fork-session分支出会话而不影响原会话

代码审查

/review在会话中审查 Pull Request
claude ultrareview云端多 Agent 深度代码审查
claude ultrareview 123审查指定 PR #123
claude ultrareview main对比 main 分支审查

Hooks 自动化

Hooks 在特定生命周期事件触发自定义脚本，配置在 settings.json 中。

事件触发时机
PreToolUse工具执行前
PostToolUse工具执行后
NotificationClaude 发送通知时
StopClaude 完成回复时
settings.json 
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "echo '即将执行 Bash 命令'"
      }]
    }]
  }
}

Vibe Coding 最佳实践

先写 CLAUDE.md

编码规范、构建命令、架构笔记——让 Claude 开工前就了解项目

提示要具体

"给 UserForm 的 email 字段加输入验证" 比 "修一下表单" 好 10 倍

用 @ 引用文件

精确指向相关文件，避免 Claude 猜测上下文

先写测试再实现

让 Claude 先写测试，再实现功能，然后 "运行测试修复失败"

用 /compact 省 token

对话变长时运行 /compact，压缩上下文保留关键信息

-c 继续对话

用 claude -c 无缝继续上次的工作，不用重新解释上下文

关键文件速查

文件/目录用途
~/.claude/CLAUDE.md全局指令（所有项目生效）
<project>/CLAUDE.md项目级指令
~/.claude/settings.json全局设置（模型、权限、Hooks、MCP）
<project>/.mcp.json项目 MCP 服务器配置
~/.claude/memory/MEMORY.md跨会话持久记忆
~/.claude/skills/已安装的 Skills 目录
~/.claude/plugins/已安装的插件目录
~/.claude/sessions/保存的会话记录
